Introduction

This page contains a list of all tools available for physics analysis. If a tool is listed on this page it does not mean that the tool is official, but it does mean that the package author thinks the tool is useful beyond his own analysis and that he is willing to provide at least some level of support.

The plan is that for the most relevant tools, we will put together a short usage example and possibly feature them in future PAT tutorials. Relevant is a very subjective term, but if you want to vote a particular tool in you can send me an email and I'll think about it.

If you have questions about a particular tool please contact the authors, or follow the instructions they posted for the case of problems. In general it is good form to include the name of the tool in the subject line. If you find the information on this page incorrect or out-of-date or misleading, or if you have any other feedback on this page, please contact Nils.Erik.Krumnack@cern.ch.

Adding More Tools

If you have a tool that you would like to add to this list, please contact Nils.Erik.Krumnack@cern.ch. It doesn't matter if this is your own tool, or just a tool that you feel might belong here. If this is your own tool, please try to answer the questions below to the best of your ability, but don't worry if you can't answer all of them.

  • Is this package still current and supported? Or has it been made obsolete by another package? If so, which one?
  • Is this package suitable for a list of general analysis tools? That doesn't mean it has to be useful for everyone, but it should be useful beyond your own analysis and you should be willing to provide at least some level of support.
  • Should this package be featured by itself, or is it part of a larger group of packages that are meant to be used together?
  • Is this package maintained as part of the official software of a physics or performance group and if so which one?
  • What is the SVN URL for this package?
  • What are the official contact email addresses for this package? A mailing list or web form is acceptable here.
  • Are there any specific instructions the user should follow before/when contacting you about problems?
  • Where can the user find documentation for this package?
  • Apart from the documentation, are there any examples of how to use this package?
  • Can you give a short summary of what this package does?

Missing ET

MissingETUtility

description: METUtility provides assistance for recomputation of the MET from analysis objects that reproduces the treatment from the MET algorithms. This has two applications. Firstly, post-reconstruction corrections and calibrations applied to the analysis objects can be accounted for in the MET. Secondly, uncertainties on these objects can be propagated in a consistent manner. The uncertainties on MET in many cases stem directly from the uncertainties on the input analysis objects. However, due to various corrections, cell overlap-removal etc, the contribution of the object momentum to the MET sum may not be 100%. METUtility accounts for this when propagating uncertainties from objects to MET so as not to inflate or deflate the impact. Uncertainties will be propagated with a size consistent with the influence of the object on the global MET, to a close approximation.

status: supported
type: dual-use
group: jet-etmiss

documentation: https://twiki.cern.ch/twiki/bin/viewauth/AtlasProtected/MissingETUtility

contacts: teng.jian.khoo@cern.ch, caterina.pizio@mi.infn.it

SVN URL: atlasoff/Reconstruction/MissingETUtility

Trigger

TrigAnalysisInterfaces + TrigBunchCrossingTool

description: TrigAnalysisInterfaces and TrigBunchCrossingTool should always be treated together. Neither of them can live without the other one. They provide detailed access to the fill pattern of the LHC. This includes the positions and intensities of all the bunches in the machine at a given time. The user can figure out how bunches were surrounding the colliding bunch of a given event using this code. This is mostly used by the non-beam backgrounds and the jet groups. Pileup behaviour is the most important for them. But there's nothing that would prevent any other analysis from using this code. It's just that physics-wise the mentioned two areas are the most affected by this.

status: supported
type: dual-use
group: trigger

documentation: https://twiki.cern.ch/twiki/bin/viewauth/Atlas/TrigBunchCrossingTool

example: /afs/cern.ch/user/k/krasznaa/public/bunch

contacts: hn-atlas-TriggerHelp@cern.ch

SVN URL: atlasoff/Trigger/TrigAnalysis/TrigBunchCrossingTool

status: supported

TileTripReader

Overlap tools

AssociationUtils: ASG recommendations for overlap removal in dual-use xAOD tools.

TopCommonInputs (OverlapTagging): Specific to the TopCommonInputs analysis framework, the tool flags overlapping reconstructed objects after object selection.

Trigger tools

TriggerAnalysisTools: Information about access to TriggerDecisionTool, trigger-reconstructed object matching, and bunching crossing tools (in Athena and AthenaROOTAccess).

Metadata tools

CutFlowSvc: ongoing project to organize writing and reading of DPD bookkeeping information (for example, what event filters have been applied to this file and how many events pass each cut).

Truth tools

MCTruthClassifier: Match reconstructed object to truth objects, and classify the truth object based on its origin (for example a true electron from a parent W boson).
FlavorTagTruthJets: Determine the flavor of a true jet (ex. b-jet, c-jet, tau-jet).
PDFReweightingTool

Embedding Tools

EmbeddingTools: The general (long term) idea of the EmbeddingTools package is to select muons or electrons in data events and replace them by simulated tau decays. Currently runs on ESDs.

Dumping AOD containers

Dump AOD containers: Prescriptions from the Physics Workbook about how to dump the container names in an AOD file (release 15).

Requests for new tools

  • Generic overlap removal tools: make a selection, check/tag for overlap, re-reconstruct jets, MET, etc. framework independent .
  • Metadata bookkeeping tools: read and write simulation metadata (for example does the sample contain pileup).
  • Metadata bookkeeping tools: read and write (sub-detector) data quality flags and selections of these flags.
  • More uniform way of adding new object identification (for example like Robust electrons).

Tools for ROOT (or D3PD) Analysis

Data handling tools

FileStager: Tool for running over several files of a given collection (ex. (d)AODs, (d)ESDs, DPDs) on nearby Tier2 and Tier3 sites, or castor.

MC reweighting tools

PileupReweighting

Tool for finding out where in a bunch train your event is

TrigBunchCrossingTool

Performance bookkeeping tools

InsituPerformance: A tool for derivation, application, monitoring, and storage of performance measurements (for example object efficiencies and resolutions). There are InsituPerformance packages for egamma, muons, taus, trigger reweighting, inner detector track, and MET. Performance results can be generated in Athena (typically by the respective combined performance group), and saved into root files or COOL (ongoing development) files, which can be later read by users in Athena or ROOT.

Fitting tools

KinFitter package: General kinematic fitting package with non-linear constraints.

Statistical tools

RooFit: Package within ROOT to allow users to built models based on real or simulated data.
RooStats: Statistical package in ROOT, built on top of RooFit to apply statistical tests to models (built in RooFit).

Trim D3PDs on the grid

EGammaTipsAndTricks: Example from EGamma group of how to trim (reduce variable content) D3PDs over the grid.

B-tagging calibration tool

BtaggingCalibrationTool: Gives a b-tagging scale factor and associated uncertainties, on jet-by-jet and/or event-by-event basis.

Build package for dual use and root-only packages

RootCore: package under development to simplify the build of packages.

Sample management

SampleHandler: package under development for managing/accessing data samples

Event loop management

EventLoop: package under development for managing the event loop

Plot Creation

MultiDraw: package under development for filling histograms using text formulas.

Tools for AthenaROOTAccess Analysis

Trigger tools

TriggerAnalysisTools: Information about access to TriggerDecisionTool, trigger-reconstructed object matching, and bunching crossing tools (in Athena and AthenaROOTAccess).

Other Tools

* CernVM-ATLAS tutorial: Run Athena on MAC or Windows
