ATLAS Standard Model Electroweak Group

Convenors: Despina Sampsonidou and Tairan Xu (atlas-phys-sm-ewk-subconveners@cern.ch)

This working group coordinates the collaboration among ATLAS members who are interested in Electroweak Physics and provides the strategies and associated tools needed for the analyses. Current measurements of interest for this group involve di-bosons (Wγ/Zγ, WW, WZ, and ZZ final states) and their interactions (for example setting limits on anomalous trilinear gauge couplings) as well as vector-boson scattering (VBS) and vector-boson fusion (VBF) processes and tri-boson physics (VVV, VVγ, Vγγ), which can be used to study quadratic gauge boson couplings. Other electroweak analyses like the Z forward-backward asymmetry and exclusive production of two bosons are also included.

Editorial board request procedure

The more often you present in the subgroup, the smoother the edboard request procedure will go!

New for Release >=22 analyses: Before EB request, the PerformanceKickOffWork will have to be signed off by the CP groups. This might involve a presentation in their meeting

Timeline:

  • Step 0: 2-4 weeks prior to target edboard subgroup presentation date, send a working draft of the INT note to the subgroup conveners. Now mandatory!
    • We will ensure nothing major is missing or provide a list of what needs to be completed before the edboard request can proceed
  • Step 1: at least one week prior to target edboard subgroup presentation date, upload the completed draft of your INT note to cds and send a link to the subgroup conveners
    • We will read and comment on the INT note and either provide a list of what needs to be completed or schedule an edboard request presentation at the next subgroup meeting
    • CP and PMG contacts will review the basic inputs used in physics analyses please include the relevant information in suitable appendices so that it is easy to find.
  • Step 2: Analysis presents a full summary in a subgroup meeting.
    • If no issues are flagged, proceed to step 3
  • Step 3: The SM conveners request an editorial board for the analysis

Example: Moriond edboard request deadline is 9 Dec (Monday). Last subgroup presentation possibility is 6 Dec, so INT note must be completed and have passed through one review by the SMEW subconveners by 29 November.

What does complete look like?

  • The supporting document must contain at least:
    • Executive Summary: motivation; physics goal and general signal characteristics; overall analysis strategy including what is new; outline of background estimation method including general characteristics of CRs/ VRs/SRs; list of outstanding tasks with a rough timeline.
    • Data and MC: a full list of background samples, details of signal samples, and a description of the dataset being used. The analysis release, full name of the samples (including p-tags) should be listed. Additionally, an outline of the blinding strategy (contact the SMEW subconveners if you are not sure) must be included, if relevant.
    • Object selection and detailed event selection: This should be final, and clearly described in the document.
    • Backgrounds: All backgrounds should be evaluated. The document should include control and validation region plots with the full dataset that will be used in the analysis.
    • Systematics: All main systematic uncertainties should be included, and their effect on the final result evaluated.
    • Results: An overview of the final fit setup including the final discriminating variables(s), the (SR/CR) regions to be included in the fit and the floating normalisation parameters. Some rough first expected limits/discovery sensitivity plots are useful if you have them but not necessary. In this case, the binning of the final variable(s) and the systematics smoothing/pruning should be indicated.
    • Unfolding: The final unfolding setup must be described, including the final discriminating variable(s) that will be shown. For blinded analyses the binning should be defined based on the expected number of events, and for non-blinded analyses binning should be defined. The background and systematic uncertainties for the differential distributions should be evaluated.
  • Additionally, you will need to complete:
    • List of contributions: this should be uploaded into CDS and synced to glance (This is important in order to avoid to appoint edboard members of the same institute of the analysers or that authors are suggested as edboard members)
    • Glance entry: The auxiliary metadata must be filled in, and in the left sidebar, as many boxes as possible should be filled from the following: "Statistical Interpretation", "Statistical Tools", "ML/MVA Tools", "Triggers". Please notify the SMEW subconveners if you have issues with this step.
    • Appointment of internal note editors and contact editors: This will be done by the SM conveners, in consultation with the analysis team.
    • List of suggested EB members: A list of at least six potential EB members, following the guidelines on the EditorialBoardrequest twiki, should be sent to the SMEW subconveners.

see this presentation at PC for more details, from PubComHome themselves, as well as the EditorialBoardrequest twiki

Archival of analysis code

  • At the time of arXiv submission, all code that is required to go from derivations to plots in the paper, final fits, and extracted cross-sections must be archived in the SM EW gitlab archive
    • Please make sure that the correct tag/version of the code is archived. If multiple branches are in the gitlab repository, please make sure that the correct branch is pushed.
    • Please make a best-effort attempt to document the necessary environment to run the code (e.g. analysis release version or root version, any external packages). This information can be included in a readme or similar file in the repository.
    • Please include a list of data/MC derivations that are processed (or, if centrally-produced ntuples/CxAODs/?? are used, document the version of these instead).

  • The following instructions provide the recommended method to archive code:
    • First, create as many repositories you need within the subgroup already created for your analysis (i.e. the appropriate STDM-20XX-XX folder in https://gitlab.cern.ch/atlas-physics/sm/ew/archive/).
    • To push your code to the appropriate locations, first ensure that all changes are committed to the working branch, then run the following two commands. Here, YOUR_BRANCH_NAME is the name of the branch you wish to push to the remote (most likely master): 
        git remote add archive https://:@gitlab.cern.ch:8443/atlas-physics/sm/photons-jets/archive/stdm-2017-33/stdm-2017-33-histo.git
        git push archive **YOUR_BRANCH_NAME**

Please note that you must have maintainer privileges to push to a fresh repository in this manner. All contact editors listed in glance have maintainer privileges. If you see the following error message when pushing 

        login:LundUnfolding mleblanc$ git push -u archive master
        Counting objects: 267, done.
        Delta compression using up to 4 threads.
        Compressing objects: 100% (251/251), done.
        Unable to rewind rpc post data - try increasing http.postBuffer
        error: RPC failed; result=65, HTTP code = 401
        fatal: The remote end hung up unexpectedly
        Writing objects:  75% (201/267), 1.05 GiB | 24.17 MiB/s

Try to run git config http.postBuffer 524288000, and then try pushing again.

Differential Cross-Section Measurement - Unfolding

  • Unfolding technique is used to correct the detector smearing effect from observed distributions and go back to fiducial region
  • So far (before 2016), most electroweak analyses have been using the Iterative Bayesian Method implemented in RooUnfold package
    • Most have used the EWUnfolding interface (originated by Matthias Schott) that handle input histograms/ntuples and interact with standard RooUnfold tools to derive unfolding results. This is now being developed as VIPUnfolding by Valerie Lang and Stefan Richter. Gitlab: VIPUnfolding.
    • Slides describing VIPUnfolding: link to pdf.
    • If you find any bugs or are interested in further developing the interface, please contact the electroweak conveners and/or the package maintainers.
  • There are some other tools used in other ATLAS analyses
    • An indepth discussion (with information) in the SM twiki page: link
    • If you are interested in working with additional tools, you are encouraged to contact and discuss with the electroweak conveners
