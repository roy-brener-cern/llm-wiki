Projects

DPD

Proposal for the "DPD Train"

DPD Making

  • DPDMakingTools: Introduction to making DPDs.
  • D2PD Maker: Instructions for making D2PDs (DAODs/DESDs).
  • D3PD Maker: Instructions for making D3PDs.
  • AODFix: For release 16, if need to recalculate quantities from AOD to produce DPDs.

DPD Reading

  • D3PDReader: A package that develops a skeleton framework for a given D3PD.
  • D3PD content: List of available D3PD packages.
  • D3PD variable request
  • RootCore: A new package that checks out, compiles, and creates libraries for many common tools used in D3PD analysis (ex. Good Run List).
  • Mana: A tool to do very fast D3PD analysis in a collaborative manner with C++ and/or python on your laptop, grid,... (Tutorial)

DPD Software Validation

  • D3PDMakerTagSubmission: Tests required to validate changes to D3PD packages before submission to the tag collector.

DPD Content/Production Validation

  • DPD validation: Tool that nighlty collects changes in DPDs, sends warning mails and displays DPD performance information.

Long Shutdown 1 Projects

  • LS1 Deliverables: List of projects/deliverables to be completed in Long Shutdown 1. Not including the Analysis Model Task Forces.
  • AMSG Task Forces: Description of the 3 task forces commissioned by the OAB.
    • "Format Task Force": The task force meant to develop the necessary technologies for upgrading the analysis EDM such that it would be usable with a minimal overhead over ROOT.
    • "Reduction Framework Task Force": The task force meant to develop the grid infrastructure for efficiently creating user/group datasets out of the centrally produced AOD datasets.
    • "Analysis Framework Task Force": The task force meant to develop a uniform environment for analysers for using the tools provided by the combined performance groups.
  • xAOD Format Task Force: A small group of people, composed of, but not limited to members of Task Force 1 and Task Force 3, defining the format of the new analysis EDM.
